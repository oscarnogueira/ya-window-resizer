# Yet Another Window Resizer

A macOS Stream Deck plugin that positions and resizes the frontmost window from
your keys — with a **configurable gap** between the window and the screen edge,
and between adjacent windows.

It’s built from scratch on the official `@elgato/streamdeck` SDK plus a small
native addon over the macOS Accessibility API. Nothing is scripted through
AppleScript or `yabai`.

## Why

Existing window movers snap a window flush to the screen edge. This one lets you
keep a **breathing gap** you control — the feature that motivated the project.

## Actions

| Action | What it does |
| --- | --- |
| **Position** | Move/resize the window to a preset: halves, thirds (incl. two‑thirds), quarters/corners, maximize, or **center** (the middle 3×3 of a 5×5 grid). |
| **Custom** | Free placement — enter `x / y / width / height` in **percent** or **pixels**. |
| **Cycle Corners** | Each press steps the window clockwise through the four corners. |
| **Cycle Sides** | Toggle the window between the left and right halves. |
| **Cycle Top/Bottom** | Toggle the window between the top and bottom halves. |

Cycle actions remember their position **per button**.

## The gap (offset)

Every position action has two settings in its Property Inspector, both defaulting
to **4 px**:

- **Screen gap** — space between a window edge and the screen edge.
- **Window gap** — space between two adjacent windows. Applied as `windowGap / 2`
  per internal edge, so two windows meeting at a split sum to a full `windowGap`
  between them.

Custom placement is literal (no gap logic). The **accent color** used by the key
icons is a single global setting.

Geometry is computed against each display’s usable area (`visibleFrame`, which
already excludes the menu bar, notch, and Dock) and always targets the window on
the screen it currently occupies.

## Requirements

**To run:**

- macOS 11 or later
- Stream Deck app 6.5 or later
- **Accessibility permission** granted to the Stream Deck app
  (System Settings → Privacy & Security → Accessibility)

**To build:**

- Node 20+ and Xcode Command Line Tools (`xcode-select --install`) for the
  native addon
- [`librsvg`](https://formulae.brew.sh/formula/librsvg) (`brew install librsvg`)
  only if you regenerate the icons
- The Elgato CLI (`npm i -g @elgato/cli`) only for `npm run pack`

## Build & install

```bash
npm install
npm run build          # native addon + JS bundle
```

Then link it into Stream Deck and (re)start the app — `./dev.sh` does the build,
first‑run link, and restart in one step:

```bash
./dev.sh
```

Or manually:

```bash
ln -sf "$(pwd)/fyi.oz.yet-another-window-resizer.sdPlugin" \
  "$HOME/Library/Application Support/com.elgato.StreamDeck/Plugins/fyi.oz.yet-another-window-resizer.sdPlugin"
```

The first time you press a key, grant Accessibility to the Stream Deck app.

Package a distributable `.streamDeckPlugin`:

```bash
npm run pack
```

### Layout

```
src/
  geometry/        pure, fully unit-tested placement math (the offset lives here)
  icons/           procedural SVG generators for the key/sidebar icons
  native/          typed wrapper around the addon (the single rounding boundary)
  actions/         Position, Custom, and the three Cycle actions
  settings.ts      settings types + gap resolution
  plugin.ts        entry point
native/src/window.mm             Objective-C++ N-API addon (Accessibility API)
fyi.oz.yet-another-window-resizer.sdPlugin/   the packaged plugin (manifest, ui, imgs)
```

Build artifacts (`bin/plugin.js`, `window.node`) are generated by the build and
git‑ignored; the repo holds source only.

## How it works

The key press flow: read the frontmost window → pick its screen → compute the
target frame from the position and gaps → apply it via the Accessibility API.

The target window is resolved from **CGWindowList** (the frontmost on‑screen
window’s owner), a live window‑server query. `NSWorkspace.frontmostApplication`
is deliberately avoided as the primary source — it updates through AppKit
activation notifications that a Node process never pumps, so it reports a stale
app and would move the wrong window.

Key icons are generated procedurally as SVG and set at runtime with `setImage`,
so they reflect the selected position and the accent color live.
